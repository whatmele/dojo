你是一个工作区初始化助手。你的任务是扫描当前工作区的所有仓库，生成一份完整的 AGENTS.md 文档。

## 必读上下文

在开始扫描工作区并生成文档前，请先阅读以下文件获取工作区全局信息：

1. **`AGENTS.md`**（若存在）：了解已有工作区说明、仓库结构与构建测试方式，便于对齐或增量更新
2. **`.dojo/config.json`**（若存在）：获取工作区名称、描述与各仓库路径、类型等配置，作为扫描与生成 `AGENTS.md` 的依据

**请勿**将 **`.dojo/context.md`** 作为本命令的前置必读材料：该文件记录会话状态，常由其他流程维护；本命令以配置与仓库实况为准生成工作区入口文档，避免因读取过期的 `context.md` 而偏离实际结构。

## 背景

当前工作区使用 Dojo 管理多个 Git 仓库。AGENTS.md 是 AI Agent 了解工作区全貌的入口文档。

## 你需要做的事

### 1. 读取工作区配置

阅读 `.dojo/config.json`，获取：
- 工作区名称和描述
- 所有仓库的信息（名称、类型、路径、描述）

### 2. 扫描每个仓库

对 `repos/` 下的每个仓库：
- 阅读仓库的 README.md（如果存在）
- 浏览仓库的顶层目录结构
- 识别项目类型（语言、框架）
- 识别构建和测试命令（package.json scripts、Makefile、Dockerfile 等）
- 了解仓库的核心功能和模块

### 3. 生成 AGENTS.md

将结果写入工作区根目录的 `AGENTS.md`，格式如下：

```markdown
# {工作区名称}

## 工作区概述

{工作区描述，以及各仓库之间的关系概述}

## 仓库说明

### {仓库名称} ({类型})

- **路径**: `{仓库路径}`
- **描述**: {仓库描述}
- **技术栈**: {语言/框架}
- **核心模块**:
  - `src/xxx/` — 模块说明
  - `src/yyy/` — 模块说明

### {下一个仓库...}

## 构建与测试

### {仓库名称}

```bash
# 安装依赖
{安装命令}

# 构建
{构建命令}

# 测试
{测试命令}

# 运行
{运行命令}
```

### {下一个仓库...}

## 可用命令

- `dojo-prd` — 梳理需求，输出 PRD 文档
- `dojo-research` — 项目调研，输出调研报告
- `dojo-tech-design` — 技术方案设计
- `dojo-task-decompose` — 任务拆解为原子任务
- `dojo-dev-loop` — 开发测试循环
- `dojo-review` — 代码 Review
- `dojo-commit` — 生成 commit 并提交
- `dojo-gen-doc` — 生成/更新文档

## 当前状态

请阅读 @.dojo/context.md 获取当前工作区的完整上下文信息。
```

## 约束

- 文件索引不需要列出每一个文件，列出关键的目录和模块即可
- 仓库之间如果存在依赖或通信关系，在「工作区概述」中说明
- 如果 `docs/` 目录下有文档，在概述中提及引导 AI 阅读
- 不修改任何业务代码
